# 🔧 Fix Redis Connection Error on Render

## ❌ Problem
Your app is trying to connect to Redis at `localhost:6379` but needs to connect to Render's Redis service.

**Error:** `Redis Client Error: connect ECONNREFUSED ::1:6379`

## ✅ Solution: Add Redis Environment Variable

### Step 1: Get Redis Internal URL from Render

1. Go to **Render Dashboard**: https://dashboard.render.com
2. Click on **cerebrexia-redis** (your Redis service)
3. Look for **"Internal Redis URL"** or **"Connection Info"**
4. Copy the URL - it looks like:
   ```
   redis://red-xxxxxxxxxxxxx:6379
   ```

### Step 2: Add REDIS_URL to Your App

1. Go back to **Render Dashboard**
2. Click on **cerebrexia-app** (your web service)
3. Click **Environment** tab
4. Click **"Add Environment Variable"**
5. Add:
   - **Key:** `REDIS_URL`
   - **Value:** Paste the Internal Redis URL you copied (e.g., `redis://red-xxxxxxxxxxxxx:6379`)
6. Click **"Save Changes"**

### Step 3: Wait for Redeploy

- Render will automatically redeploy (2-3 minutes)
- Watch the logs - Redis errors should disappear
- App should start successfully

## 🎯 Quick Copy-Paste Instructions

### If You Can't Find the Redis URL:

**Option A: Use Render's Auto-Generated Variable**

Render automatically creates environment variables for linked services. Try adding:

**Key:** `REDIS_URL`  
**Value:** `${cerebrexia-redis.REDIS_URL}`

This tells Render to use the internal connection string automatically.

---

**Option B: Manual Connection (If Option A doesn't work)**

1. In **cerebrexia-redis** service, find the **Internal Connection String**
2. It will be in format: `redis://red-xxxxxxxxxxxxx:6379`
3. Copy that exact URL
4. Add it as `REDIS_URL` in cerebrexia-app environment variables

## 📋 Complete Environment Variables Checklist

After adding Redis URL, your app should have these environment variables:

```
✅ DATABASE_URL (auto-generated by Render)
✅ REDIS_URL (you need to add this)
✅ RAZORPAY_KEY_ID (dummy: rzp_test_dummy123456789)
```

## 🚨 Troubleshooting

### If Redis URL doesn't work:

**Try this format instead:**
```
redis://default:password@red-xxxxxxxxxxxxx:6379
```

### If you still see connection errors:

1. Make sure **cerebrexia-redis** service is **"Available"** (green status)
2. Check that both services are in the **same region**
3. Verify the Redis URL is correct (no typos)

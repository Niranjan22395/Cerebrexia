# 🔧 Fix Razorpay Error on Render

## ❌ Error
```
Error: `key_id` or `oauthToken` is mandatory
at new Razorpay (/app/node_modules/razorpay/dist/razorpay.js:34:13)
```

## 🎯 Root Cause
The application is trying to initialize Razorpay without API keys. The environment variables `RAZORPAY_KEY_ID` and `RAZORPAY_KEY_SECRET` are not set on Render.

## ✅ Solution - Add Environment Variables

### Option 1: Add Real Razorpay Keys (Production)

1. **Go to Render Dashboard**
2. Click on **cerebrexia-app** service
3. Go to **Environment** tab
4. Click **Add Environment Variable**
5. Add these two variables:

```
RAZORPAY_KEY_ID=rzp_test_YOUR_KEY_ID
RAZORPAY_KEY_SECRET=YOUR_SECRET_KEY
```

**Get Razorpay Keys:**
- Go to https://dashboard.razorpay.com
- Sign up/Login
- Go to Settings → API Keys
- Generate Test Keys
- Copy Key ID and Secret

### Option 2: Add Dummy Keys (For Testing Without Payment)

If you just want to test the app without payment functionality:

1. **Go to Render Dashboard**
2. Click on **cerebrexia-app** service
3. Go to **Environment** tab
4. Add these dummy values:

```
RAZORPAY_KEY_ID=rzp_test_dummy_key_12345
RAZORPAY_KEY_SECRET=dummy_secret_key_67890
```

**Note:** Payment features won't work with dummy keys, but the app will start.

## 🚀 After Adding Variables

1. **Save** the environment variables
2. Render will **automatically redeploy** your service
3. Wait 2-3 minutes for deployment
4. Check logs - should see: `✅ Server running on port 5000`

   - Go to App Passwords
   - Generate password for "Mail"

## 🐛 Still Having Issues?

### Check Logs for Errors:
```bash
# Look for these in Render logs:
✅ "Server running on port 5000" - Good!
❌ "key_id is mandatory" - Need to add Razorpay keys
❌ "Connection refused" - Database issue
❌ "ECONNREFUSED" - Redis issue
```

### Common Fixes:

1. **Razorpay Error:**
   - Add `RAZORPAY_KEY_ID` and `RAZORPAY_KEY_SECRET`

2. **Database Error:**
   - Check `DATABASE_URL` is set
   - Verify cerebrexia-db is running

3. **Redis Error:**
   - Check `REDIS_URL` is set
   - Verify cerebrexia-redis is running

4. **Email Error (Non-critical):**
   - App will still work without email
   - Add `EMAIL_USER` and `EMAIL_PASSWORD` later
